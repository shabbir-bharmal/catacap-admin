Release 2026-05-19

Migrations

1. 001_cover_fees_escrow_reconciliation.sql
   - Schema:
       * NEW TABLE cover_fees_escrow_reconciliation_log
             (id serial PK, sponsor_user_id varchar(450) UNIQUE,
              reconciled_amount numeric(14,2), reconciled_at timestamp,
              details jsonb)
       * NEW INDEX cover_fees_escrow_reconciliation_log_sponsor_idx
             on (sponsor_user_id)
   - Intent:
       Reconcile the historical double-debit on escrow-backed Cover Fees
       pools (task #522). Prior to this release, escrow pools
       (reserved_amount > 0) debited the sponsor wallet BOTH at funding
       time AND on every donor fee draw — funds were already in escrow,
       so the per-donor debit was a second debit on the same money. This
       migration credits each sponsor back the cumulative
       still-unreversed per-activity fee debits + the canceled-activity
       fee debits (the cancel path never returned funds to the wallet)
       across every escrow pool they sponsor, and writes an
       account_balance_change_logs entry per sponsor.
   - Idempotency:
       Each sponsor's credit is gated on no existing row in
       cover_fees_escrow_reconciliation_log for that sponsor_user_id;
       a re-run is a no-op once a sponsor has been reconciled. Schema
       objects are CREATE ... IF NOT EXISTS.
   - Transactional: BEGIN/COMMIT.
   - Rollback: commented at the bottom of the migration. For each row
     in cover_fees_escrow_reconciliation_log, subtract
     reconciled_amount back from users.account_balance, then DROP the
     log table.
